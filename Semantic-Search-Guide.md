# GravBlog Semantic Search Guide

Welcome to the new Semantic Search feature in GravBlog! This guide explains what it is, how it works, and how you can get the most out of it.

## 1. What is Semantic Search?

Traditional search engines rely on exact keyword matches. If you search for "automobile", a traditional search won't find articles that only use the word "car" or "vehicle".

Semantic search solves this by understanding the *meaning* and *context* of your query. It uses Artificial Intelligence (AI) to map both your search phrase and the blog posts into a mathematical space called a "vector space". Concepts that have similar meanings reside close to each other in this space.

In GravBlog:
- Searching for "**Styling modern UI**" will successfully find posts about "**Tailwind CSS**".
- Searching for "**User authentication systems**" will find posts about "**NextAuth.js**".
- Searching for "**Fast databases**" will point to articles about "**Neon** or **Prisma**".

## 2. How to Use It (For Users)

1. Open your GravBlog website in the browser (e.g., `http://localhost:3000`).
2. Locate the **Search bar** in the top navigation menu.
3. Instead of typing an exact title, try typing a concept, full phrase, or abstract idea.
   - Example abstract search: *"How do I make my website look good?"*
4. Press Enter. You'll be taken to the `/search` page where your query is transformed into a vector and instantly matched against the database!

## 3. How it Works (Under the Hood)

GravBlog now features a 100% local, free semantic intelligence pipeline using the following stack:

- **Transformers.js**: A local AI model (`Xenova/all-MiniLM-L6-v2`) runs directly inside your Next.js backend. It turns text into a 384-dimensional array of numbers (an embedding). *Note: No external API keys (like OpenAI) are required, protecting your privacy and saving costs!*
- **Pgvector**: A powerful extension for your Neon PostgreSQL database, capable of storing high-dimensional vectors.
- **Prisma ORM**: The database client executes raw PostgreSQL queries using the L2 distance operator (`<->`) to instantly find the closest mathematical matches out of thousands of records.

## 4. Creating New Content

You don't need to do anything special to index new posts!

Whenever you create a new post (or update an existing one) via the Admin Panel, GravBlog automatically intercepts the save event. It takes the post's title, excerpt, and content, feeds them into the AI model, and calculates the vector embedding. This is then saved seamlessly alongside the post. Within milliseconds, your new content becomes fully searchable by concept.
